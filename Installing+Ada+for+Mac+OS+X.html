Installing Ada on Mac OS X (Mountain Lion or later)

To install Ada on your Mac, perform the following steps:

1) Go to http://libre.adacore.com/download/configurations and, in the middle of the page, where it says "Select your platform", choose "x86-64 Mac OS X (64 bits)" and "GNAT GPL 2013" (yes, 2013 not 2014)

2) Expand the portion labeled "GNAT Ada GPL 2013 " and click on the link that says " gnat-gpl-2013-x86_64-apple-darwin12.2.0-bin.tar.gz" to download and save the file on your computer. Then, uncompress the downloaded file.

3) In a shell (e.g. terminal or under emacs), navigate down into the resulting directory until you see the files README, doinstall, etc.  Then, type,

sudo ./doinstall

You will be asked for the administrator password (which is likely just your password on your Mac).  You should accept the default directory in which the files are to installed.

4)  Once the installation has completed (and the files are in /usr/local/gnat), you will need to add "/usr/local/gnat/bin" to the front of the PATH environment variable.  To do this, you can put one of the following lines in your shell configuration files (e.g. .bashrc, .cshrc, etc.)

      PATH=/usr/local/gnat/bin:$PATH; export PATH  (Bourne shell)
      setenv PATH /usr/local/gnat/bin:$PATH        (C shell)

 

Testing the compiler

In the Ada resources folder, click on "Simple Ada program to test compiler".  A zip file will be downloaded.  When you unzip it, you will see several files with .ads and .adb extensions.  In a shell, go to the directory containing the files. Then type,

gnatmake objmain.adb

You will see that gcc is invoked several times, along with gnatbind and gnatlink.

Note:  If, the first time you try compiling an Ada program using "gnatmake", you get an error complaining about a missing library, be sure that your command line tools for OS X are up to date.   Go to https://developer.apple.com/downloads/ and download and install the "Command Line Tools" for your version of Mac OS X.  Then, try running "gnatmake objmain.adb" again.

To run the program, type

./objmain

(you won't need the "./" if your PATH environment variable contains ".")
